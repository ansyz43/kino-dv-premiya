# Деплой на reg.ru (shared-хостинг, PHP)

## Что собираем

```
cd astro
npm install
npm run build
```

Готовая статика лежит в `astro/dist/`.
Внутри есть `dist/api/lead.php` — PHP-обработчик формы заявки.

## Куда заливать

1. Зайти в **Хостинг → Файловый менеджер** (или FTP/SFTP).
2. Открыть корневую папку домена — обычно `public_html/` или `httpdocs/`.
3. Залить **всё содержимое `astro/dist/`** в этот корень
   (так, чтобы `index.html` лежал прямо в `public_html/`).
4. Файл `public_html/api/lead.php` отработает форму — никаких
   дополнительных настроек не нужно: на reg.ru shared-хостинге PHP включён
   по умолчанию, и встроенная функция `mail()` сразу шлёт письма через
   локальный sendmail.

## Куда падают заявки

Файл `api/lead.php` отправляет письма на:

```
dvfilmaward2026@mail.ru
```

Поменять адрес можно прямо в файле (вверху, константа `MAIL_TO`).

## Откуда уходят (важно для антиспама)

По умолчанию `From: noreply@<домен сайта>`. Когда домен будет привязан в
панели reg.ru:

- Создайте mailbox `noreply@вашдомен.ру` в почте reg.ru.
- В `lead.php` константу `MAIL_FROM` можно явно поставить
  на этот ящик — письма будут проходить SPF/DKIM проверки mail.ru
  без попадания в «Спам».

## Лог заявок

На случай, если `mail()` упадёт (квоты хостинга, временный сбой),
все заявки **также пишутся** в текстовый лог:

```
public_html/api/leads.log
```

Файл создаётся автоматически. Просматривайте через файловый менеджер
панели reg.ru — формат «дата | IP | имя | e-mail | телефон | название».

## Локальный тест

`vite dev` НЕ исполняет PHP — форма в dev-режиме вернёт 404 на
`/api/lead.php` и покажет fallback-сообщение «напишите на
dvfilmaward2026@mail.ru». Это нормально. Реальная проверка — на
проде после заливки `dist/`.

## Быстрый прод-чек

После заливки откройте напрямую:

```
https://вашдомен.ру/api/lead.php
```

Должно ответить JSON: `{"ok":false,"error":"Method not allowed"}` —
значит PHP запустился и обработчик жив.
